# 脚本マーケット 実装ガイド

全機能を最初から完成形で実装する。フェーズ分けは依存関係に基づく実装順序であり、MVPではない。

---

## フェーズ1: 基盤構築

### Step 1: プロジェクト初期化
- Next.js プロジェクト作成（App Router、TypeScript）
- Tailwind CSS + shadcn/ui のセットアップ
- Noto Sans JP フォントの導入
- トンマナの適用（globals.css にカラー変数設定）
- ディレクトリ構成の作成（`docs/spec.md` セクション3に準拠）
- ESLint + Prettier の設定

### Step 2: Firebase セットアップ
- Firebase プロジェクト作成（開発用）
- Firebase Authentication 有効化（Google + メール/パスワード）
- Firestore データベース作成
- Firebase Storage 有効化
- `lib/firebase.ts`（クライアント）、`lib/firebase-admin.ts`（サーバー）の初期化
- 環境変数の設定（`.env.local`）

### Step 3: 認証・ユーザーID
- アカウント登録画面（Google認証 + メール/パスワード）
- ログイン画面
- ユーザーID設定画面（初回ログイン時リダイレクト）
- `(app)/` 配下の認証ミドルウェア
- ユーザーID未設定時のリダイレクトロジック
- メール確認フロー（メール/パスワード認証の場合）
- `users` コレクションの作成ロジック
- プロフィール編集画面

### Step 4: 共通レイアウト
- ヘッダー（スマホ: ハンバーガーメニュー、PC: ナビゲーション）
- フッター
- レスポンシブ対応の確認

**テストチェックポイント1:**
- Google認証でログインできること
- ユーザーID設定画面が表示されること
- ユーザーIDを設定後、マイページに遷移すること
- メール/パスワードでも登録・ログインできること
- ヘッダー・フッターが表示されること

## フェーズ2: 台本出品・表示

### Step 5: Stripe Connect セットアップ
- Stripe Connect Express のオンボーディングフロー
- `stripe.ts` / `stripe-client.ts` の初期化
- Stripe Webhook エンドポイント（`api/webhooks/stripe/route.ts`）
- `account.updated` Webhook処理

### Step 6: 台本出品機能
- 出品画面（PDFアップロード、全メタデータ入力、特性タグ選択）
- PDFのFirebase Storageへのアップロード
- スラッグ生成ロジック（タイトルから自動生成、重複チェック）
- `scripts` コレクションへの保存
- バージョン管理（`scripts/{id}/versions/`）
- 出品管理画面（一覧、編集、PDF差し替え、非公開化）

### Step 7: 台本詳細ページ
- 台本詳細ページ（`(public)/scripts/[scriptId]/`）
- 全メタデータの表示（あらすじ、キャスト、上演料テーブル、上演履歴、作家コメント）
- 特性タグ・バッジの表示
- 短縮URL（`/scripts/{id}`）からスラッグ付きURLへのリダイレクト
- SEO対応（メタタグ、JSON-LD、OGP）
- ユーザープロフィールページ（`(public)/users/[userId]/`）

### Step 8: PDFプレビュー
- pdf.js によるブラウザ上プレビュー
- 冒頭5ページ制限（6ページ目以降はブロック）
- 全文無料公開台本の場合は全ページ表示
- ページ送り、拡大縮小

**テストチェックポイント2:**
- Stripe連携（テストモード）が完了できること
- 台本を出品できること（PDF、メタデータ、特性タグ）
- 台本詳細ページが正しく表示されること
- プレビューが冒頭5ページのみ表示されること
- 出品後に編集・PDF差し替えができること
- ユーザープロフィールページに出品台本が表示されること

**テストチェックポイント3:**

## フェーズ4: チャット・上演許可

### Step 12: チャット基盤
- チャットルーム開設（`startChat`。相手1人につき1ルーム）
- リアルタイムメッセージ送受信（Firestore onSnapshot）
- チャット一覧画面
- チャットルーム画面（LINE風UI）

### Step 13: ヒアリングシート・相談
- デフォルトヒアリングシート設定画面
- 台本個別ヒアリングシート設定
- ヒアリングシート回答画面（「上演許可の相談をする」→ 回答 → `createConsultation` 実行）
- `createConsultation`: consultationsドキュメント作成、チャット開設（既存があればそこへ）、systemメッセージ＋hearingSheetResponseメッセージ投稿
- 回答のチャット内カード表示
- 作家へのメール通知

### Step 14: 都度請求
- 請求作成（チャット内「請求を作成する」ボタン。関連する相談の選択は任意）
- 請求カード表示（金額、ラベル、「支払う」ボタン）
- 請求支払い（Stripe Checkout）
- Webhook処理（invoices.status更新）
- 請求のキャンセル
- システムメッセージ（支払い完了、キャンセル）

### Step 15: 相談ステータス管理
- consultations コレクションのstatusの自動遷移（作家の初回返信で未対応→対応中）
- 「やり取りを完了する」ボタン（`completeConsultation` 呼び出し）
- マイページの相談管理画面（作家側・利用者側。データソースはconsultationsコレクション）
- consultationCount の自動カウント

**テストチェックポイント4:**
- 「上演許可の相談をする」→ ヒアリングシート → consultations作成＋チャット開設の一連のフローが動作すること
- 同じ相手との2回目以降の相談が既存チャットルームに入ること
- チャットでリアルタイムにメッセージが送受信できること
- 請求の作成→支払い（テストモード）→完了が動作すること
- 相談管理画面でステータスが正しく表示されること
